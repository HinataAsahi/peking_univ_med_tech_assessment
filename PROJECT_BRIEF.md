# ECS Paper-to-ARM Agent 项目说明

## 1. 项目背景

本项目面向脑细胞外间隙（brain extracellular space, ECS）领域。

我的目标有两个：

1. 系统学习 ECS 的基础概念、生理机制、研究方法和疾病诊疗背景。
2. 在七日考核时间内完成一个真实可运行、可检查、可复现的 Agent 原型。

## 2. 选择方向

选择考核 Track A：Paper-to-ARM 科学资产。

项目暂定名称：

ECS Paper-to-ARM Agent

## 3. 核心输入

- 至少 5 篇真实 ECS 相关论文；
- 论文 PDF 或解析后的文本；
- 必要时使用人工确认的论文元数据。

## 4. 核心输出

每篇论文生成结构化 ARM，至少包含：

- metadata
- claims
- evidence
- protocol
- runbook
- eval_plan
- provenance
- limitations
- artifacts

## 5. 必须满足的考核要求

- 支持至少 5 篇真实论文；
- 提取不少于 5 条关键结论；
- 每条结论能够定位到原文段落、页码、图或表；
- 模型推断与论文原文明确区分；
- 至少一个方法步骤能够安全试运行；
- 支持校验、回放和导出；
- 展示一个成功案例；
- 展示一个失败、证据不足或解析错误案例；
- 提供运行命令、自动测试、README 和已知限制；
- 展示至少 3 条：
  调研发现 → 设计决定 → 实现结果。

## 6. 七日范围限制

优先完成可靠的最小闭环，不追求功能数量。

第一版计划：

- 一个主 Agent；
- 两到三个工具；
- 一种 Pydantic 结构化输出；
- JSON/YAML 导出；
- 一个简单网页或命令行入口；
- 自动校验与运行日志。

## 7. 初步工具

- PDF 文本与页码提取工具；
- Claim-Evidence 对齐工具；
- DOI/PMID 或引用校验工具；
- ARM Schema 校验工具；
- 一个安全的 dry-run 工具。

## 8. 暂时不做

- 不自动复现完整湿实验；
- 不做临床诊断；
- 不建立大型通用脑科学知识图谱；
- 不先开发复杂多 Agent 系统；
- 不支持所有论文排版格式；
- 不保证自动识别所有复杂图表。

## 9. 学习目标

系统理解：

- ECS 的定义与组成；
- ECS volume fraction 和 tortuosity；
- ECS 中扩散与物质运输；
- ECS 的测量、示踪和成像方法；
- ECS 与脑水肿、肿瘤、神经退行性疾病等的关系；
- ECS 对药物递送和脑内清除的影响。

## 10. 技术原则

- 论文原文优先于模型推断；
- 每条重要输出必须保留 provenance；
- 缺少证据时不得补写；
- 不能准确定位原文时标记 review_required；
- 先完成单 Agent 闭环，再考虑多 Agent；
- 所有重要功能必须有测试。
