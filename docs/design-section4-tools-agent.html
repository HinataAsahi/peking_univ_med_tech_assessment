Tool 1：parse_pdf

纯函数 · 无 LLM

接口签名

from agents import function_tool @function_tool def parse_pdf(file_path: str) -> ParsedPaper: """解析 PDF 论文，提取带页码标注的结构化文本。 Args: file_path: PDF 文件的绝对路径 Returns: ParsedPaper: 包含 pages (文本+页码) 和 metadata Raises: IngestError: PDF 损坏、无文本层或页数超限 """ import fitz doc = fitz.open(file_path) pages = [] for i, page in enumerate(doc): blocks = page.get_text("blocks") text = "\n".join( b[4] for b in blocks if b[6] == 0 # block_type == text ) pages.append(Page( number=i + 1, text=text, blocks=[...] )) return ParsedPaper( pages=pages, metadata=_extract_metadata(doc), raw_text="\n\n".join(p.text for p in pages) )

被调用方式

# Orchestrator 在 Ingest 阶段调用 def run_ingest(paper_path: Path) -> ParsedPaper: result = parse_pdf(str(paper_path)) if not result.pages: raise IngestError("no_text_layer") return result

Token 管理策略

风险：全文一次性输入可能超过 DeepSeek V4 上下文窗口。

缓解：
  • 先传摘要 + 标题 + 关键词做第一轮提取
  • 如果 claims 不够 5 条或 LLM 要求更多上下文，逐章传入
  • 最终组装所有分章结果为完整 ExtractionResult

错误处理

情况Agent 行为
LLM 返回 claims < 5二次扫描未被覆盖的章节，仍不足则在 limitations 中记录"论文结论密度低"
某 claim 缺 evidenceextraction_method 设为 review_required，不编造
LLM API 超时/限流重试 3 次（指数退避），仍失败则终止并保留已提取的部分结果
返回的 quoted_text 不在原文中Extract 阶段不拦截，由 Validate 阶段检测并标记

沙箱实现

import math from typing import Any # 白名单：只允许这些内置函数和模块 _ALLOWED_BUILTINS = { "abs": abs, "round": round, "min": min, "max": max, "sum": sum, "len": len, "range": range, "list": list, "float": float, "int": int, "True": True, "False": False, "None": None, } _ALLOWED_MODULES = {"math": math} def safe_exec(expr: str, params: dict) -> float: """在白名单沙箱中执行 Python 表达式""" safe_globals = { "__builtins__": _ALLOWED_BUILTINS, **_ALLOWED_MODULES, **params } safe_locals = {} try: result = eval(expr, safe_globals, safe_locals) return float(result) except Exception as e: raise SandboxError(str(e))

Tool 4：validate_arm

纯函数 · 无 LLM

接口签名

@function_tool def validate_arm( arm: ARM, paper: ParsedPaper # 用于 hallucination 检测 ) -> ValidationReport: """校验 ARM 的完整性、一致性和可追溯性。 6 条校验规则（详见第三节）： 1. Schema 完整性（Pydantic 类型校验） 2. Claim ≥ 5, Evidence ≥ 5 3. 每条 Claim 关联 ≥ 1 Evidence 4. Hallucination 检测（quoted_text vs 原文） 5. Provenance 覆盖率 6. 模型推断不冒充原文 Args: arm: 待校验的 ARM 对象（部分或完整填充） paper: 原文（用于规则 4 的文本匹配） Returns: ValidationReport: 含 passed bool + 所有违规项 """

6 条规则实现摘要

规则实现阻塞级别
R1: Schema 完整性ARM.model_validate(arm) 捕获 ValidationErrorCRITICAL
R2: 数量下限len(claims) >= 5 and len(evidence) >= 5CRITICAL
R3: Claim-Evidence遍历 claims，检查 evidence_refs 非空且指向存在的 evidenceCRITICAL
R4: Hallucination精确子串匹配 + fuzzy token overlap (threshold=0.6)CRITICAL
R5: Provenance递归遍历 ARM 所有字段，检查有 source 的字段占比WARNING
R6: 推断标记检查所有 extraction_method == "llm_inferred" 的字段是否已明确标记CRITICAL

3. 预留接口：Agentic Self-Check 循环 v1.1 规划

设计原则：v1.0 用确定性 Pipeline 确保七日交付。但架构在 Validate 阶段预留了循环接口，逻辑验证后可以无缝升级为"校验失败 → 回到 Extract 重新提取"的 Agentic 模式。

预留接口定义

class PipelineState(BaseModel): """跨阶段的全局状态（v1.0 用，v1.1 扩展）""" paper_path: str parsed: ParsedPaper | None = None extraction: ExtractionResult | None = None dry_runs: list[DryRunResult] = [] # v1.1 新增 retry_count: int = 0 # 当前重试次数 max_retries: int = 3 # 最大重试次数 last_validation: ValidationReport | None = None def should_retry(self) -> bool: """判断是否需要 agentic 重试""" if self.retry_count >= self.max_retries: return False if self.last_validation is None: return False # 仅对 hallucination 和 claim-evidence gap 重试 has_fixable = ( self.last_validation.hallucination_flags or self.last_validation.claim_evidence_gaps ) return has_fixable
