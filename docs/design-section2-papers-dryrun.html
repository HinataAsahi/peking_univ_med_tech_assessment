  • 数学公式提取不完整 → LLM 在 Extract 阶段根据上下文推断公式语义

综述

论文 1：ECS 结构与生理参数综述

搜索词：brain extracellular space review volume fraction tortuosity 2023-2025

预期来源：Physiological Reviews / Nature Reviews Neuroscience / Trends in Neurosciences
覆盖目标：ECS 定义、α 概念、λ 概念、测量方法总览
定量要求：包含 α 和 λ 在不同脑区的典型值表
计算验证：λ = √(D/D*) 定义式；α 在不同脑区的对比
# Dry-run 示例 input: D_free = 7.6e-6 cm²/s, λ_reported = 1.6 calc: D_star = D_free / λ² = 7.6e-6 / 2.56 = 2.97e-6 cm²/s paper: D*_reported ≈ 2.9e-6 result: |2.97 - 2.9| / 2.9 = 2.4% → ✓ PASS
方法学

论文 2：ECS 扩散参数测量方法

搜索词：ECS diffusion measurement iontophoresis integrative optical imaging 2023-2024

预期来源：Journal of Neuroscience Methods / NeuroImage / eLife
覆盖目标：ECS 测量方法、示踪剂、扩散方程应用
定量要求：包含浓度-时间数据或可拟合的扩散参数
计算验证：扩散方程拟合；α 和 λ 的交叉验证
# Dry-run 示例：扩散方程拟合 model: C(t) = (Q/α)·(4πD*t)^(-3/2)·exp(-r²/4D*t - k't) input: 浓度-时间数据点（从论文表格提取） fit: 最小二乘法拟合 → α_fitted, λ_fitted check: α_fitted vs α_reported, λ_fitted vs λ_reported
方法学/成像

论文 3：ECS 结构成像与示踪剂扩散

搜索词：brain ECS imaging super-resolution microscopy tracer diffusion 2023-2025

预期来源：Nature Communications / Science Advances / eLife
覆盖目标：ECS 成像方法、示踪剂尺寸与扩散关系
定量要求：包含扩散系数或 tracer 分布数据
计算验证：Stokes-Einstein 方程；示踪剂半径 vs 扩散系数
# Dry-run 示例 input: T = 310 K, η = 0.001 Pa·s, r = 10 nm calc: D_pred = kT / (6π η r) = (1.38e-23 × 310) / (6 × π × 0.001 × 1e-8) = 2.27e-11 m²/s paper: ECS 中实测 D_eff ≈ 2.0e-12 m²/s result: D_eff << D_pred → ECS 几何约束显著 ✓
疾病关联

论文 4：脑水肿/缺血状态下 ECS α 的动态变化

搜索词：brain extracellular space ischemia edema volume fraction 2023-2025

预期来源：Stroke / JCBFM / Brain
覆盖目标：ECS 与脑水肿、脑缺血的病理关联
定量要求：包含正常 vs 病理状态的 α 或扩散参数对比
计算验证：扩散时间在不同 α 下的变化
# Dry-run 示例：扩散时间对比 input: 扩散距离 x = 100 μm, D*_normal = 2.0e-6 cm²/s calc: t_normal = x² / (2 × D*) = 25 s D*_ischemia ≈ D*_normal / 3（α 崩塌导致） t_ischemia = 75 s result: 扩散延迟 3 倍 → 与论文描述的"slowing"一致 ✓

论文 5：类淋巴系统与 ECS 脑内清除

搜索词：glymphatic brain clearance perivascular transport drug delivery 2023-2025

7. 案例演示脚本

✅ 成功案例

场景：综述论文，验证 λ = √(D/D*) 关系
## 论文原文段落 "In the cortex, the tortuosity λ was found to be approximately 1.6, with D = 7.6×10⁻⁶ cm²/s in free medium and D* ≈ 2.9×10⁻⁶ cm²/s in the extracellular space." ## Agent 行为 1. Extract → claim "皮层 λ ≈ 1.6 (D=7.6e-6, D*=2.9e-6 cm²/s)" 2. Compute → 识别公式 λ = √(D/D*) 3. Dry-run → sqrt(7.6e-6 / 2.9e-6) = 1.618 4. Compare → |1.618 - 1.6| / 1.6 = 1.1% < 5% → passed ## 演示台词 "Agent 不仅提取了结论，还自动识别了其中隐含的计算关系， 用代码验证了论文数据的一致性——误差仅 1.1%。"

❌ 失败案例

场景：疾病论文，声明了 α 的相对变化但缺少绝对参数
## 论文原文段落 "Following the induction of ischemia, the extracellular volume fraction decreased significantly, reaching approximately 30% of its baseline value within 5 minutes." ## Agent 行为 1. Extract → claim "缺血后 α 降至基线的约 30%" 2. Compute → 尝试 dry-run： 需要参数：baseline_α, ischemia_α, time_course 可获取：仅 relative_change=30%，无绝对值 3. 结果 → insufficient_data → review_required: "缺少基线 α 绝对值和完整时间序列" ## 演示台词 "Agent 识别出论文提出了定量结论但缺少验证所需的参数， 正确地标记了 review_required 而非编造数据。 这就是 ARM 的核心价值——区分'论文说了什么'和'我们能验证什么'。"
